# 章节复习
## 绪论 
### 算法概念
1. **算法定义**：算法是指解决问题的一系列计算步骤，是解决方案准确而完整的描述。
2. **算法的基本性质**：
    - 输入性：0或多个输入。
    - 输出性：至少1个输出。
    - 确定性：指令确定，无歧义。
    - 有限性：有限步内结束。
    - 有效性：用纸笔就能算出。
3. **算法设计的基本步骤**：
    - 分析求解问题
    - 建立数学模型
    - 选择算法、数据结构
    - 描述算法
    - 证明算法正确性
    - 算法分析

### 算法分析
1. **时间复杂度**：算法运行时间与输入规模的增长关系。
2. **空间复杂度**：算法运行所需的内存空间与输入规模的增长关系。

**渐进符号**

大$O$：紧凑上界（最常用）  
大$\Omega$：紧凑下界  
大$\Theta$：紧凑上下界（最严格）

**最好情况、最坏情况、平均情况**

**分析时间复杂度**
- 非递归
1. 找出层数最多的循环。
2. 分析循环内语句的执行次数。
3. 用渐进符号表示循环内语句的执行次数。
    > 例：若为k重循环，每个循环执行n次，则时间复杂度为$O(n^k)$。
- 递归
1. 分析递归过程。
2. 建立运行时间的递推关系式。
3. 根据递归出口求解递推表达式。
4. 用渐进符号表示时间复杂度。
    > 例1：二分查找。
    > 1. 分析递归过程：每次递归会调用2次搜索范围减半的子递归和一个常数时间的比较和赋值操作。
    > 2. 建立运行时间的递推关系式：$T(n) = 2T(n/2) + O(1)$，递归出口：$T(1) = O(1)$。
    > 3. 根据递归出口求解递推表达式：$T(n) = O(logn)$。
    > 4. 用渐进符号表示时间复杂度：$O(logn)$。

    > 例2：汉诺塔。
    > 1. 分析递归过程：每次递归会调用2次盘片数量-1的子递归和1次盘片从A移动到C的操作。
    > 2. 建立运行时间的递推关系式：$T(n) = 2T(n-1) + 1$，递归出口：$T(1) = O(1)$。
    > 3. 根据递归出口求解递推表达式：$T(n) = 2^n - 1$。
    > 4. 用渐进符号表示时间复杂度：$O(2^n)$。

**分析空间复杂度**
- 非递归
1. 找出函数所有临时变量（形参不考虑，因为引用传递）。
2. 分析变量的空间之和。
3. 用渐进符号表示空间复杂度。
- 递归
递归除了临时变量，还要考虑调用栈的大小，故分析过程和时间复杂度相似，**都是建立递推关系式**，**求解递推表达式**，**用渐进符号表示空间复杂度**。

## 递归和分治
- [递归](01.递归/recursion.md)
- [分治](01.递归/divide_and_conquer.md)
### 基本概念
1. **递归定义**：函数定义中调用自身的**方法**。（直接或间接）
2. **分治定义**：将问题分解为多个规模较小的子问题，**递归解决**子问题，然后合并子问题的解得到原问题的解，是一种算法设计**策略**。
3. **递归特点**:
    - 结构相同、参数不同。
    - 调用次数有限。
    - 存在结束递归条件（递归出口）。
4. **分治特点**：
    - **可分解性**：问题可分解为几个子问题求解。
    - **可缩性**：问题缩到一定程度即可直接解决。
    - **可合成性**：可由子问题的解合成原问题的解。
    - **独立子问题**：子问题之间相互独立，子问题的解不影响其他子问题的解。
5. **分治基本步骤**：
    - 分解：将原问题分解为若干个规模较小的子问题。
    - 解决：递归地求解各子问题。
    - 合并：将各子问题的解合并为原问题的解。
6. **递归与分治的关系**：分治是一种递归的特殊形式，所有分治算法都是递归算法，但并非所有递归算法都是分治算法。

### 典型例题
- [阶乘函数](01.递归/recursion.md#1.阶乘函数)
```cpp
return n == 0? 1 : n * factorial(n-1);
```
时间复杂度：$O(n)$  
空间复杂度：$O(n)$
- [汉诺塔](01.递归/recursion.md#4.汉诺塔)
```cpp
// 递归出口
if(n == 1){
    move(1, a, c); return;
}
hanoi(n-1, a, c, b);        // 递归分解
move(n, a, c);              // 移动最底端盘子
hanoi(n-1, b, a, c);        // 递归分解
```
时间复杂度：$O(2^n)$  
空间复杂度：$O(n)$
- [归并排序](01.递归/recursion.md#6.归并排序)
```cpp
if(b >= e) return;         // 递归出口
mergesort(arr, b, f);      // 递归分解
mergesort(arr, f+1, e);    // 递归分解
merge(arr, b, f, f+1, e);  // 合并2数组
```
时间复杂度：$O(nlogn)$  
空间复杂度：$O(n)$

- [快速排序](01.递归/recursion.md#7.快速排序)
```cpp
if(b >= e) return;         // 递归出口
flag(arr, b, e);           // 划分子数组
quicksort(arr, b, f);      // 递归分解
quicksort(arr, f+1, e);    // 递归分解
```
时间复杂度：$O(nlogn)$  
空间复杂度：$O(logn)$
> 快速排序的合并是隐式的，因为flag保证子数组有序。

## 回溯法
### 基本概念
1. **回溯法定义**：具有限界函数的深度优先生成法称为回溯法。
2. **回溯法基本思想**：
  - 深度优先搜索
  - 只保存根到当前结点路径，在叶子结点（一个解）或死结点（无解）回溯
  - 剪枝（剪死结点和非最优解）
3. **回溯法 VS 穷举法**：
    - 相同点：都是对整个解空间搜索。
    - 不同点：回溯法可以剪枝。
4. **回溯法基本步骤**：
    - 子集树：当所给问题是从n个元素的集合S中找出满足某种特性的子集时，解空间为S的幂集。
    - 代表问题：0-1背包、装载问题
    ```cpp
    int a[n]; // 状态变量
    void backtrack(int m)
    {
        if(m > n) output(); // 递归结束条件
        else 
        {
            a[m] = 0;         // 设置状态，0表示不要该物品
            if(...) backtrack(m + 1); // 递归搜索，继续搜索下一个物品
            a[m] = 1;         // 设置状态，1表示要该物品
            if(...) backtrack(m + 1); // 递归搜索，继续确定下一个物品
        }
    }
    ```
    - 排列树：当所给问题时从n个元素的集合S中找出满足某种性质的排列时，使用排列树。解空间为S的全排列。
    - 代表问题：八皇后、素数环
    ```cpp
    void backtrack(int t)
    {
        if(t > n) output(); // 递归结束条件
        else
        {
            for(int i = t; i <= n; i++)
            {
                swap(x[t], x[i]); //交换x[t]和x[i]
                if(legal(t)) backtrack(t + 1); // 如果m处可放置，则搜索下一层
                swap(x[t], x[i]); // 交换x[t]和x[i]
            }
        }
    }
    ```

## 分支限界法
### 基本概念
1. **分支限界法定义**：具有限界函数的广度优先（队列）或以最大收益优先（优先队列）生成法称为分支限界法。
2. **分支限界法基本思想**：
  - 广度优先搜索（最大收益优先搜索）
  - 每个结点只能成为1次拓展结点。
  - 剪枝：剪不符合限界函数的结点。
>[!important]
> 分支限界法限界函数的设计非常重要！  
> 因为同层次搜索时，需要限界函数指导搜索方向，进而更快找到最优解。  
> 如果设计不好，可能会耗时巨大，甚至错过最优解。
1. **分支限界法 VS 回溯法**：

| **算法**       | **解空间搜索方式** | **存储结点的数据结构** | **结点存储特性**                       | **常用应用**                                         |
| -------------- | ------------------ | ---------------------- | -------------------------------------- | ---------------------------------------------------- |
| **回溯法**     | **深度优先搜索**   | **栈**                 | **只保存从根结点到当前扩展结点的路径** | **能够找出满足约束条件的所有解**                     |
| **分支限界法** | **广度优先搜索（或最大收益优先）**   | **队列或者优先队列**   | **每个结点只有一次成为活结点的机会**   | **找出满足约束条件的一个解或者满足目标函数的最优解** |
4. **分支限界法基本步骤**：
```cpp
typedef struct node
{
    int status; // 状态变量
    int bound;  // 限界函数值

    node(int _status); // 构造函数
    get_bound(int _status); // 计算限界函数值

    bool operator < (const node& n) const; // 优先队列比较函数
};

void search()
{
    queue<node> q; / priority_queue<node> q;    // 队列
    q.push(nd0); // 根结点入队
    while(!q.empty())
    {
        node nd = q.top(); q.pop(); // 取出队首结点
        ... // 生成扩展结点
        for(node nd1 : expand(nd))
        {
            if(is_target(nd1)) 
              output(vi); break;        // 找到解
            if(notused(vi)) q.push(vi); // 未使用过的结点
        }
    }
}
```

## 动态规划
- [点击跳转](05.动态规划/dp.md)

**基本思想**

将带求解问题分解成若干个子问题，先求解子问题，然后**从这些子问题的解**得到原问题的解。

**基本步骤**

1. 找出最优解的性质与结构
2. 递归定义最优解
   - 即设计dp数组
3. 自底向上计算最优解（子问题推大问题）
   - 即状态转移方程
4. 根据计算结果构造最优解

**基本性质**

最优子结构性质、重叠子问题性质、无后效性。

- **最优子结构性质**：最优解包含的子问题解也是最优的。（反证法证明）

- **重叠子问题性质**：子问题不总是新问题。自底向上时，可将子问题解保存在dp表中，这样只需计算1次。

- **无后效性**：子问题解一旦确定就固定不动了。

## 贪心
- [点击跳转](06.贪心/greed.md)

**基本思想**

将大问题分解为子问题，将**子问题局部最优解**合成**全局最优解**。

**基本步骤**

1. 建立数学模型来描述问题。
2. 把求解的问题分成若干个子问题。
3. 对每一子问题求解，得到子问题的局部最优解。
4. 把子问题的局部最优解合成原问题的最优解。

**基本要素**

最优子结构性质、贪心选择性质

- **最优子结构性质** ：最优解包含的子问题解也是最优的。（反证法证明）

- **贪心选择性质** ：整体最优解可通过局部最优解得到。（数学归纳法证明）

## 概率算法
- **基本要素**：
1. **不可再现性**：无法复刻上一次结果。
2. **算法分析困难**：无法分析算法。

# 总结
## 基本思想：
- **分治**：将一个难以直接解决的大问题，分解成规模较小的相同子问题，直至这些子问题容易直接求解，并可利用这些子问题的解求出原问题的解。
- **动态规划**：和分治法类似，将原问题分解成多个子问题，子问题之间往往不相互独立，保留已解决的问题的子问题，避免重复计算
- **贪心**：总是找出当前最优解，局部最优解。
- **回溯、分支限界**：从初始状态出发，搜索能达到是所有状态；当一条路走到尽头，则回退一步或多步，从另一个状态继续出发，直到所有路径都搜索过。

## 基本性质：
- **重叠子问题**：动态规划
- **最优子结构**：贪心、动态规划
- **多米诺性质**：回溯、分支定界
- **独立子问题**：分治
- **贪心选择性质**：贪心

## 基本步骤：
- **分治法**：分解、解决、合并
- **回溯法、分支限界法**：定义解空间、判断多米诺性质、设计限界和剪枝、搜索
- **动态规划**：定义状态、设计dp数组、自底向上计算、构造最优解
- **贪心**：选择合适的贪心策略、证明贪心选择性质、构造最优解
