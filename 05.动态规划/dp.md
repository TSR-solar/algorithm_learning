# 动态规划
- [概念](#基本思想) 
  1. 最优子结构、重叠子问题、无后效性
  2. 设计dp、状态转移、结果表示
- [一重dp所有状态最值](#最大连续子序列和)
   1. 状态i下最值
   2. 继续选或者抛弃
   3. 结果为数组里最大（小）的
- [二重dp最后状态](#三角形最小路径)
   1. 状态i, j下最值（有时需预留一行）
   2. 所有入口求min或max
   3. 结果为最终状态
   4. 可从dp数组找到最优解具体状态
- [图dp](#旅行商问题)
   1. 状态V、i下最值（V为结点集合，用二进制位表示）
   2. 所有V内结点求最值
   3. 结果为最终状态
- [区间dp](#矩阵连乘问题)
   1. 区间i到j下最值
   2. 枚举划分点k
   3. 结果为dp[0][n-1]
  
## 基本思想
将带求解问题分解成若干个子问题，先求解子问题，然后**从这些子问题的解**得到原问题的解。

## 基本步骤
1. 找出最优解的性质与结构
2. 递归定义最优解
   - 即设计dp数组
3. 自底向上计算最优解（子问题推大问题）
   - 即状态转移方程
4. 根据计算结果构造最优解

## 基本要素
最优子结构性质、重叠子问题性质、无后效性。

### 最优子结构性质
最优解包含的子问题解也是最优的。（反证法证明）

### 重叠子问题性质
子问题不总是新问题。自底向上时，可将子问题解保存在dp表中，这样只需计算1次。

### 无后效性
子问题解一旦确定就固定不动了。

## 与分治法的异同
### 联系
基本思想都是分解为子问题，从子问题的解得到原问题的解。

### 区别
动态规划因为记录子问题的解，所以可以避免分治的重复计算。

## 最大连续子序列和
### 问题描述
给定一个含n（n>=1）个整数的序列，要求求出其中最大连续子序列的和。

### 输入输出
输入：第一行输入n；  
第二行输入n个整数，代表序列。
```
12
-6 2 4 -7 5 3 2 -1 6 -9 10 -2
```
输出：最大连续子序列和。
```
16
```

### 问题求解
1. **设计dp**：dp设为一维数组，`dp[i]`表示以第`i`个元素结尾的最大连续子序列和。
2. **状态转移方程**：
   ```
   dp[0] = a[0], i=0（初始条件）
   dp[i] = max(dp[i-1] + arr[i], arr[i]), i>0
   ```
3. **结果表示**：找出dp中**最大的一个**，即为最大连续子序列和。

### 代码实现
**[maxsum.cpp](代码/maxsum.cpp)**

### 算法分析
时间复杂度：$O(n)$ **一重循环**

空间复杂度：$O(n)$ **一维数组dp**

## 最长递增子序列
### 问题描述
给定一个无序的整数序列`a[0..n-1]`，求其中最长递增（严格）子序列的长度。

### 输入输出
输入：
```
9
2 1 5 3 6 4 8 9 7
```
输出：
```
5
```

### 问题求解
1. **设计dp**：dp设为一维数组，`dp[i]`表示以第`i`个元素结尾的最长递增子序列的长度。
2. **状态转移方程**：
   ```
   dp[i] = 1, 0 <= i <= n-1 (初始条件)
   dp[i] = max{dp[j] + 1}, a[j] < a[i] && j < i, 0 <= i <= n-1
   ```
3. **结果表示**：找出dp中**最大的一个**，即为最长递增子序列的长度。

### 代码实现
**[maxinclen.cpp](代码/maxinclen.cpp)**

### 算法分析
时间复杂度：$O(n^2)$ **二重循环**

空间复杂度：$O(n)$ **一维数组dp**

## 三角形最小路径
### 问题描述
给定一个高度为n的整数三角形，求从顶部到底部的最小路径和及其一条最小路径，从每个整数出发只能向下移动到相邻的整数。

### 输入输出
输入：
第一行一个正整数，表示高度n  
之后每行从1到n以1递增，表示三角形。
```
4
2
3 4 
6 5 7 
8 3 9 2
```
输出：
```
13
```

### 问题求解
1. **设计dp**：dp设为二维数组，`dp[i][j]`表示从底部到`a[i][j]`的最小路径和。
2. **状态转移方程**：  
   - `dp[i][j] = a[i][j]`，`i == n-1` 
   - `dp[i][j] = min(dp[i+1][j], dp[i+1][j+1]) + a[i][j]`，`0 < i < n-1`  
3. **结果表示**：`dp[0][0]`为最小路径和。

### 代码实现
**[triangle.cpp](代码/triangle.cpp)**

### 算法分析
时间复杂度：$O(n^2)$ **二重循环**

空间复杂度：$O(n^2)$ **二维数组dp**

## 最长公共子序列
### 问题描述
给定两个字符串 `s1` 和 `s2`，求它们的最长公共子序列（LCS）。

### 输入输出
输入：两个字符串 `s1` 和 `s2`。
```
abcbdb
acbbabdbb
```

输出：第一行：`s1` 和 `s2` 的最长公共子序列长度。  
第二行：`s1` 和 `s2` 的最长公共子序列。
```
5
acbdb
```

### 问题求解
1. **设计dp**：dp设为二维数组，`dp[i][j]`表示`s1[0..i-1]` 和 `s2[0..j-1]` 的 LCS 的长度。  
   > [!caution]
   >
   > 注意`dp[i][j]`对应的是`i-1`和`j-1`！  
   > 因为对应`i`和`j`的话，状态转移方程会变得复杂。
2. **状态转移方程**：
- `dp[i][j] = 0`，`i == 0 || j == 0`
- `dp[i][j] = dp[i-1][j-1] + 1`，`s1[i-1] == s2[j-1]`
- `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`，`s1[i-1] != s2[j-1]`
3. **结果表示**：`dp[m][n]`为`s1` 和 `s2` 的 LCS 的长度。
4. **求LCP**：
   1. 首先令`i = m`，`j = n`。
   2. - 若`dp[i][j]=dp[i-1][j]`，移到上一行，`a[i-1]/b[j-1]`不是LCS字符。
      - 若`dp[i][j]=dp[i][j-1]`，移到左一列，`a[i-1]/b[j-1]`不是LCS字符。
      - 若`dp[i][j]=dp[i-1][j-1]+1`，移到左上方，`a[i-1]/b[j-1]`是LCS的字符。
   3. 重复此操作，直到`i=0`或`j=0`，得到LCS。


### 代码实现
**[lcs.cpp](代码/lcs.cpp)**

### 算法分析
时间复杂度：$O(mn)$ **二重循环**，`m`和`n`分别是`s1`和`s2`的长度。

空间复杂度：$O(mn)$ **二维数组dp**

## 编辑距离
### 问题描述
给定两个字符串`s1`和`s2`，求它们的编辑距离（最小字符操作次数）。

合法的字符操作包括：
- 插入一个字符
- 删除一个字符
- 替换一个字符

### 输入输出
输入：两个字符串`s1`和`s2`。
```
sfdqxbw
gfdgw
```

输出：编辑距离。
```
4
```

### 问题求解
1. **设计dp**：dp设为二维数组，`dp[i][j]`表示将`s1[0..i-1]` 编辑为 `s2[0..j-1]` 的编辑距离。（i-1和j-1理由同上）
2. **状态转移方程**：
   - `dp[i][j] = j`， `i = 0`（a为空串，则需要插入j个字符）
   - `dp[i][j] = i`， `j = 0`（b为空串，则需要删除i个字符）
   - `dp[i][j] = dp[i-1][j-1]`，`s1[i-1] == s2[j-1]`（末尾字符相同，不用操作）
   - `dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1`，`s1[i-1] != s2[j-1]`（末尾字符不同，取插入，删除，替换里最小的）
3. **结果表示**：`dp[m][n]`为`s1` 和 `s2` 的编辑距离。

### 代码实现
**[editdistance.cpp](代码/editdistance.cpp)**

### 算法分析
时间复杂度：$O(mn)$ **二重循环**，`m`和`n`分别是`s1`和`s2`的长度。

空间复杂度：$O(mn)$ **二维数组dp**

## 01背包
### 问题描述
给定一个背包，每个物品有重量和价值两个属性，背包最大容量为`W`，求背包能装下的物品的最大价值。

### 输入输出
输入：第一行两个整数`n`和`W`，表示物品个数和背包容量。  
之后2行，第一行输入`n`个整数，表示物品的重量`w[0..n-1]`，第二行输入`n`个整数，表示物品的价值`v[0..n-1]`。
```
5 10
2 2 6 5 4
6 3 5 4 6
```

输出：最大价值和选取的物品编号（从0开始）
```
15
0 1 4
```

### 问题求解
1. **设计dp**：`dp`设计为二维数组，`dp[i][r]`表示只考虑前i个物品并且背包容量为`r`时最大价值。
2. **状态转移方程**：
   - `dp[i][r] = 0`，`i == 0`（不考虑任何物品）
   - `dp[i][r] = 0`, `r == 0`（背包容量为0）
   - `dp[i][r] = dp[i-1][r]`，`r < w[i-1]`（但背包容量不足，放弃）
   - `dp[i][r] = max(dp[i-1][r], dp[i-1][r-w[i]] + v[i])`，`r >= w[i-1]`（背包容量足够，选择放入和不放入间的更优解）
3. **结果表示**：`dp[n][W]`为最大价值。
4. **选取物品**：
   1. 令`i = n`，`r = W`。
   2. 若`dp[i][r] = dp[i-1][r]`，不选第`i`个物品，跳到`dp[i-1][r]`。
   3. 否则`dp[i][r] = dp[i-1][r-w[i]] + v[i]`，选第i个物品，跳到`dp[i-1][r-w[i]]`。
   4. 重复此操作，直到`i=0`或`r=0`，得到选取的物品编号。

### 代码实现
**[01.cpp](代码/01.cpp)**

### 算法分析
时间复杂度：$O(nW)$ **一重循环**，`n`是物品个数，`W`是背包容量。

空间复杂度：$O(n)$ **一维数组dp**

## 旅行商问题
### 问题描述
给定n个城市和每对城市之间的距离，找到一条访问每个城市恰好一次并返回起点的最短路径。

### 输入输出
输入：第一行一个整数`n`，表示城市个数。  
之后`n`行，每行`n`个整数，表示邻接矩阵`dist[i][j]`，即城市`i`到城市`j`的距离。
```
4
0 8 36 5
6 0 8 5
8 9 0 5
7 7 8 0
```
输出：最短路径长度
```
23
```
### 问题求解
1. **设计dp**：dp[V][i]表示从顶点s出发经过V（一个顶点的集合）中全部顶点（每个顶点恰好经过一次）到达顶点i的最短路径长度。
   >[!tip]
   >
   > 为了用正整数表示一个集合，可以使用位表示的方法，其中每位表示一个顶点是否在集合中。（1或0）  
   > 例如，V = 1011，表示集合{1, 2, 4}。
2. **状态转移方程**：
   - `dp[V][i] = dist[0][i]`，`V`为空集。
   - `dp[V][i] = min(dp[V-i][j] + dist[j][i])`， `V`不为空集，`i, j`为`V`任意顶点。
3. **结果表示**：`dp[2^n-1][0]`为最短路径长度。

### 代码实现
**[tsp.cpp](代码/tsp.cpp)** 
> [!warning]
> WORK IN PROGRESS

### 算法分析
时间复杂度：$O(2^n*n^2)$， **状态数为$2^n$，每个状态需要遍历$n$个顶点，每个顶点最多访问一次，所以总时间复杂度为$2^n*n^2$**

空间复杂度：$O(2^n*n)$，**状态数为$2^n*n$**，所以空间复杂度为$O(2^n*n)$。

## 矩阵连乘问题
### 问题描述
给定n个矩阵`A1, A2, ..., An`，它们的维度分别为`p0 x p1, p1 x p2, ..., pn-1 x pn`，求矩阵连乘`A1A2...An`的最优计算顺序，使得标量乘法次数最少。

### 输入输出
输入：第一行一个整数`n`，表示矩阵个数。  
第二行n+1个整数，表示矩阵维度`p0, p1, ..., pn`。
```
6
20 25 5 15 10 20 25
```

输出：最少乘法次数。
```
15125
```

### 问题求解
1. **设计dp**：dp[i][j]表示从第i个矩阵到第j个矩阵的顺序的标量乘法次数。
2. **状态转移方程**：
   - `dp[i][i] = 0`，`i = 0, 1, ..., n-1`
   - `dp[i][j] = min(dp[i][k] + dp[k+1][j] + p[i]*p[k+1]*p[j+1]`)，`i < j`，`k`为`i`到`j`之间的分割点。
3. **结果表示**：`dp[0][n-1]`为最少乘法次数。

### 代码实现
**[matrix_chain_order.cpp](代码/matrix_chain_order.cpp)**

### 算法分析
时间复杂度：$O(n^3)$， **三重循环**，`n`是矩阵个数。前两重循环遍历dp数组，第三重循环遍历分割点。

空间复杂度：$O(n^2)$， **二维数组dp**，`n`是矩阵个数。
